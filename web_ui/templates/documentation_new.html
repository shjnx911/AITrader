Getting Started with AITradeStrategist
Welcome to AITradeStrategist

AITradeStrategist is an advanced cryptocurrency trading platform that leverages machine learning algorithms to optimize trading strategies. This guide will help you get started with the platform and understand its key features.

System Requirements
  • CPU: 4+ cores recommended
  • RAM: 8GB minimum, 16GB recommended
  • GPU: Optional but recommended for faster AI model training (AMD RX6600 or equivalent)
  • Storage: 20GB free space
  • OS: Linux, Windows, or macOS
Quick Start Guide
  1. Setup Environment

    Ensure your system meets the requirements above. For GPU acceleration, make sure you have the appropriate drivers installed.

  2. Configure API Connections

    Go to Settings > API Connections to set up your Freqtrade integration and exchange API keys.

  3. Import Historical Data

    Navigate to the Backtest page to import historical trading data for training your AI models.

  4. Train Your First AI Model

    Go to the AI Models page and click "Train New Model" to create your first trading model.

  5. Run a Backtest

    Test your model against historical data to evaluate its performance before deploying.

  6. Deploy for Live Trading

    Once satisfied with your model's performance, deploy it for live trading through the Freqtrade integration.

AI Models Documentation
Understanding AI Models in Trading

AITradeStrategist uses various machine learning algorithms to analyze market data and make trading decisions. The platform primarily supports three types of models:

Model Type Strengths Best Use Cases GPU Support
LightGBM Fast training, handles large datasets, good accuracy General-purpose trading, high-frequency strategies Yes (via DirectML)
XGBoost High accuracy, robust to overfitting Complex pattern recognition, trend following Yes
Random Forest Simple, stable, less prone to overfitting Exploratory analysis, baseline models No
Training an AI Model
  1. Prepare Data

    Import backtest data or historical market data for the cryptocurrency pairs you want to trade.

  2. Select Model Type

    Choose the appropriate model type based on your trading strategy and requirements.

  3. Configure Parameters
    Tip: For beginners, enable "Auto-optimize hyperparameters" to let the system find optimal settings.

    Key parameters to consider:

    • Feature Selection: Technical indicators used for prediction (RSI, MACD, Bollinger Bands, etc.)
    • Timeframe: The time interval for each candlestick (1m, 5m, 15m, 1h, etc.)
    • Training Period: Amount of historical data to use for training
    • Hyperparameters: Model-specific settings that affect learning behavior
  4. Train the Model

    Start the training process. The system will automatically:

    • Split data into training and validation sets
    • Preprocess and normalize features
    • Train the model with the selected algorithm
    • Perform cross-validation to ensure robustness
    • Evaluate performance metrics
  5. Review and Save

    Examine the model's performance metrics and save it if satisfactory.

Model Evaluation Metrics

When assessing model performance, consider these key metrics:

Metric Description Target Value
Win Rate Percentage of profitable trades > 60%
Profit Factor Ratio of gross profits to gross losses > 1.5
Sharpe Ratio Risk-adjusted return measurement > 1.0
Max Drawdown Maximum observed loss from a peak to a trough < 10%
Precision Accuracy of positive predictions > 0.7
Recall Ability to find all positive instances > 0.7
F1 Score Harmonic mean of precision and recall > 0.7
Best Practices
  • Avoid Overfitting: Use sufficient data and proper validation techniques
  • Regular Retraining: Update models periodically as market conditions change
  • Feature Engineering: Experiment with different technical indicators
  • Start Simple: Begin with straightforward models before adding complexity
  • Ensemble Methods: Consider using multiple models for more robust predictions
Important: While AI models can significantly improve trading performance, they are not infallible. Always monitor their performance and be prepared to intervene when necessary.
Freqtrade Integration Guide
Introduction to Freqtrade Integration

AITradeStrategist integrates seamlessly with Freqtrade, an open-source cryptocurrency trading bot. This integration allows you to deploy AI-generated strategies directly to Freqtrade for automated trading.

Setup Process
  1. Install Freqtrade

    If you haven't already, install Freqtrade following their official installation guide.

  2. Configure Freqtrade API

    Enable the REST API in your Freqtrade configuration:

    "api_server": {
    "enabled": true,
    "listen_ip_address": "127.0.0.1",
    "listen_port": 8080,
    "verbosity": "info",
    "enable_openapi": true,
    "jwt_secret_key": "your-secret-key",
    "CORS_origins": ["http://localhost:5000"],
    "username": "your-username",
    "password": "your-password"
}
  3. Connect AITradeStrategist to Freqtrade

    In AITradeStrategist, go to Settings > API Connections and configure the Freqtrade API connection:

    • API URL: The URL where your Freqtrade API is running (e.g., http://localhost:8080)
    • Username: Your Freqtrade API username
    • Password: Your Freqtrade API password
  4. Test Connection

    Click "Test Connection" to verify that AITradeStrategist can communicate with Freqtrade.

Using AI-Generated Strategies with Freqtrade
  1. Generate a Strategy

    After training an AI model, navigate to the AI Models page and click "Generate Strategy" for your selected model.

  2. Review the Strategy

    The system will generate a Python file compatible with Freqtrade's strategy format. Review the code to ensure it meets your requirements.

  3. Deploy the Strategy

    Two deployment options are available:

    • Manual Deployment: Download the strategy file and place it in your Freqtrade's "user_data/strategies" directory.
    • Automatic Deployment: Use the "Deploy to Freqtrade" button to automatically transfer the strategy to your configured Freqtrade instance.
  4. Configure Freqtrade to Use the Strategy

    Update your Freqtrade configuration to use the new strategy:

    "strategy": "AIGenerated_BTC_LightGBM_v2",
"strategy_path": "user_data/strategies/"
  5. Start Trading

    Start Freqtrade with your new strategy, or use the "Start Bot" button in AITradeStrategist's Freqtrade control panel.

Monitoring and Control

Once integrated, you can manage your Freqtrade bot directly from AITradeStrategist:

  • Dashboard: View current trades, profits, and performance metrics
  • Trade History: Access detailed records of all trades executed by Freqtrade
  • Remote Controls: Start/stop the bot, force buy/sell, and adjust parameters
  • Alerts: Configure notifications for important trading events
Data Synchronization

AITradeStrategist automatically synchronizes data with Freqtrade, including:

  • Trading Pairs: Available markets and trading pairs
  • Performance Metrics: Real-time and historical performance data
  • Trade Data: Detailed information about executed trades
  • Balance Information: Current account balance and allocation
Important: Always test your AI-generated strategies thoroughly in backtesting and dry-run mode before using them for live trading.
Backtesting Documentation
Introduction to Backtesting

Backtesting is the process of testing a trading strategy against historical market data to evaluate its potential effectiveness. AITradeStrategist provides comprehensive backtesting capabilities to help you assess and refine AI-powered trading strategies before deploying them in live markets.

Backtesting Process
  1. Data Preparation

    Begin by importing historical market data for the cryptocurrency pairs you want to test. You can either:

    • Import from Freqtrade: Use existing backtest data from your Freqtrade installation
    • Download New Data: Fetch fresh historical data for specific pairs and timeframes
  2. Strategy Selection

    Choose the trading strategy to test. Options include:

    • AI-Generated Strategies: Strategies created from your trained AI models
    • Custom Strategies: Manually created trading strategies
    • Template Strategies: Pre-configured strategies for comparison
  3. Configure Parameters

    Set the parameters for your backtest:

    • Time Range: The historical period to analyze
    • Starting Capital: Initial investment amount
    • Fee Structure: Trading fees to simulate
    • Stake Amount: Amount to invest per trade
    • Max Open Trades: Maximum number of simultaneous positions
  4. Run Backtest

    Execute the backtest. The system will simulate how your strategy would have performed during the selected historical period.

  5. Analyze Results

    Review the comprehensive results provided by the system:

    • Performance Metrics: Profit, win rate, Sharpe ratio, etc.
    • Trade List: Detailed view of each simulated trade
    • Equity Curve: Visualization of capital growth over time
    • Drawdown Analysis: Assessment of potential losses
Importing Backtest Data

To import existing backtest data from Freqtrade:

  1. Navigate to the Backtest page
  2. Click "Import from Freqtrade"
  3. Select the directory containing your Freqtrade backtest results
  4. Choose the specific backtest runs to import
  5. Click "Import Selected Data"
Tip: Imported backtest data can be used both for analysis and for training AI models.
Key Backtest Metrics
Metric Description Interpretation
Total Profit Overall profit/loss as a percentage Higher is better
Win Rate Percentage of profitable trades >50% is generally desirable
Max Drawdown Largest peak-to-trough decline Lower is better
Sharpe Ratio Risk-adjusted return measurement >1 is good, >2 is excellent
Profit Factor Gross profit divided by gross loss >1.5 indicates a potentially viable strategy
Average Trade Duration Average time positions were held Depends on strategy type
Expectancy Average profit per trade Positive values indicate profitable strategy
Using Backtest Results for AI Training

AITradeStrategist uniquely allows you to leverage backtest results to train AI models:

  1. Import backtest data as described above
  2. Navigate to the AI Models page
  3. Click "Train New Model"
  4. Select "Use Backtest Data" as the data source
  5. Choose the imported backtest results to use for training
  6. Configure model parameters and begin training

This approach allows the AI to learn from the successful patterns identified in your backtest strategies.

Important: Past performance in backtesting does not guarantee future results. Always use backtesting as one of multiple evaluation tools, and be aware of potential overfitting issues.
Monte Carlo Simulation Guide
Introduction to Monte Carlo Simulation

Monte Carlo simulation is a powerful statistical technique that uses repeated random sampling to obtain a range of possible outcomes for a given scenario. In the context of trading, it helps estimate the probability of different results by running many simulations with randomized variables.

Why Use Monte Carlo Simulation?

Monte Carlo simulations provide several key benefits for crypto traders:

  • Risk Assessment: Better understand potential downside risks and maximum drawdowns
  • Probability Analysis: Estimate the likelihood of achieving specific profit targets
  • Robustness Testing: Evaluate how strategies perform under different market conditions
  • Position Sizing: Optimize trade size based on risk tolerance
  • Expectation Management: Set realistic expectations for strategy performance
Running a Monte Carlo Simulation
  1. Navigate to the Monte Carlo Page

    Access the Monte Carlo simulator from the main navigation menu.

  2. Select a Model

    Choose one of your trained AI models for simulation. The system will use the model's historical performance metrics as a baseline.

  3. Configure Parameters
    Parameter Description Recommended Value
    Number of Simulations How many random simulation runs to perform 1,000 - 10,000
    Initial Capital Starting investment amount Your actual trading capital
    Risk Per Trade Percentage of capital risked on each trade 1% - 2%
    Trading Days Number of days to simulate 30, 90, or 365 days
    Trades Per Day Average number of trading opportunities Based on your strategy frequency
    Win Rate Baseline probability of profitable trades Use model's historical win rate
    Win Rate Variance Random fluctuation in win rate 5% - 10%
    Risk/Reward Ratio Ratio of potential profit to potential loss Use model's historical ratio
    Consider Fees Include trading fees in calculations Enabled
  4. Run Simulation

    Click "Run Simulation" to start the Monte Carlo process. The system will execute the configured number of randomized trading scenarios.

  5. Analyze Results

    Review the comprehensive results provided:

    • Equity Curves: Visualize the range of possible performance paths
    • Probability Distribution: See the distribution of final outcomes
    • Risk Metrics: Analyze drawdown probabilities and other risk factors
    • AI Analysis: Automated interpretation of results with optimization suggestions
Interpreting Monte Carlo Results

Key metrics to focus on from your simulation results:

  • Expected Return Range

    The range between the 25th and 75th percentiles represents the most likely outcome range.

  • Maximum Drawdown Probability

    The likelihood of experiencing different levels of drawdown helps with risk management.

  • Win Rate Sensitivity

    Understanding how changes in win rate affect overall profitability helps identify critical performance thresholds.

  • Profit Target Probability

    The likelihood of reaching specific profit goals within your simulation timeframe.

  • Ruin Probability

    The chance of losing a significant portion of your capital (e.g., 50% or more).

Using Results to Optimize Trading

Monte Carlo results can inform several key trading decisions:

  • Position Sizing: Adjust your risk per trade based on drawdown probabilities
  • Strategy Selection: Compare different models to find the most robust options
  • Capital Allocation: Determine appropriate capital allocation across different strategies
  • Stop-Loss Levels: Set strategic stop-loss levels based on expected price movement ranges
  • Profit Targets: Establish realistic profit expectations and targets
Pro Tip: Run multiple simulations with different parameter sets to stress-test your strategies under various market conditions. This helps identify robust strategies that perform well across a range of scenarios.
Alert System Documentation
Introduction to the Alert System

AITradeStrategist includes a powerful notification system that keeps you informed about trading activities, system events, and market conditions. The alert system can send notifications through multiple channels: Email, Telegram, and SMS.

Setting Up Notification Channels
Email Notifications
  1. Navigate to Settings > Notification Settings
  2. Enable "Email Notifications"
  3. Enter your email address
  4. Select the alert types you want to receive via email
  5. Save your settings
Telegram Notifications
  1. Create a Telegram bot using BotFather
  2. Get your bot token and chat ID
  3. In AITradeStrategist, navigate to Settings > Notification Settings
  4. Enable "Telegram Notifications"
  5. Enter your Telegram Bot Token and Chat ID
  6. Click "Test Telegram" to verify the connection
  7. Save your settings
SMS Notifications (via Twilio)
  1. Create a Twilio account at twilio.com
  2. Get your Account SID, Auth Token, and Twilio phone number
  3. In AITradeStrategist, navigate to Settings > Notification Settings
  4. Enable "SMS Notifications"
  5. Enter your Twilio Account SID, Auth Token, and phone numbers
  6. Click "Test SMS" to verify the connection
  7. Save your settings
Creating Alert Rules

To create custom alert rules:

  1. Go to the Monitoring & Alerts page
  2. Click "New Alert"
  3. Select the alert type:
    • Price Alert: Notifications when price crosses specified levels
    • Profit/Loss Alert: Alerts based on P&L thresholds
    • System Alert: Notifications for system resource usage (CPU, memory, etc.)
    • Trade Count Alert: Alerts when trade frequency exceeds thresholds
  4. Configure the specific parameters for your selected alert type
  5. Choose which notification channels to use for this alert
  6. Set the alert priority (High, Medium, Low)
  7. Save the alert rule
Alert Types
Alert Category Description Use Case
Price Alerts Notifications when price crosses above/below specified levels Identifying entry/exit opportunities, monitoring key support/resistance levels
Trade Alerts Notifications about trade entries, exits, and status changes Staying informed about bot trading activity in real-time
Profit/Loss Alerts Notifications when P&L reaches specified thresholds Managing risk, celebrating wins, addressing losses promptly
System Alerts Notifications about system resource usage and technical issues Ensuring system stability and addressing technical problems

Managing Alerts

The Monitoring & Alerts page provides tools for managing your alerts:

  • Active Alerts: View all currently active alert rules
  • Alert History: Review past alert notifications
  • Edit/Delete: Modify or remove existing alert rules
  • Enable/Disable: Temporarily activate or deactivate alerts
  • Test: Manually trigger alerts to verify notification delivery
Best Practices
  • Prioritize Alerts: Use high-priority channels (like SMS) only for critical notifications
  • Group Similar Alerts: Avoid alert fatigue by combining related notifications
  • Set Reasonable Thresholds: Ensure alert thresholds are not too sensitive
  • Test Regularly: Periodically verify that all notification channels are working
  • Use Templates: Create alert templates for common scenarios to save time

GPU Acceleration Documentation
Introduction to GPU Acceleration

AITradeStrategist leverages GPU acceleration to significantly speed up AI model training and inference. This feature is particularly beneficial for complex models and large datasets, providing up to 10x faster performance compared to CPU-only processing.

Supported Hardware

The platform is optimized for AMD GPUs through DirectML integration. Supported hardware includes:

GPU Series Supported Models Minimum VRAM Recommended VRAM
AMD Radeon RX 6000 Series RX 6600, RX 6700 XT, RX 6800, RX 6900 XT 8 GB 12+ GB
AMD Radeon RX 5000 Series RX 5700 XT, RX 5700, RX 5600 XT 6 GB 8+ GB
AMD Radeon VII All variants 16 GB 16 GB
NVIDIA GPUs are not currently supported natively, but may work through DirectML with reduced performance.
Setup and Configuration
Driver Requirements

To use GPU acceleration, ensure you have the following:

  • AMD Drivers: Adrenalin 22.5.1 or newer
  • DirectML: Automatically installed with AITradeStrategist
  • OS Support: Windows 10/11, Linux with ROCm support
Enabling GPU Acceleration
  1. Go to Settings > System
  2. Under "GPU Acceleration," ensure "Enable GPU Acceleration" is checked
  3. Optionally configure memory limits to prevent out-of-memory errors
  4. Save your settings
Memory Management

Proper GPU memory management is essential for stable operation:

  • Memory Limit: Set a maximum VRAM usage (recommended: 75% of available VRAM)
  • Batch Size Control: Adjust batch sizes when training large models
  • Model Complexity: Consider reducing model complexity if VRAM is limited
Accelerated Features

The following features benefit from GPU acceleration:

Feature Acceleration Benefit VRAM Usage
LightGBM Model Training 5-10x faster training time Medium-High
XGBoost Model Training 3-7x faster training time Medium
Model Prediction/Inference 2-4x faster prediction time Low
Feature Engineering 3-5x faster preprocessing Medium
Monte Carlo Simulation 5-8x faster simulation time Low-Medium
Hyperparameter Optimization 4-6x faster optimization High
Performance Optimization

To get the best performance from GPU acceleration:

  • Driver Updates: Keep GPU drivers updated to the latest version
  • Background Applications: Close other GPU-intensive applications when training models
  • Cooling: Ensure adequate cooling for sustained GPU workloads
  • Data Preprocessing: Perform data preprocessing on CPU before transferring to GPU
  • Batch Processing: Use appropriate batch sizes for your GPU's VRAM capacity
Troubleshooting
Issue Possible Cause Solution
Out of Memory Errors Insufficient VRAM for operation Reduce batch size, limit VRAM usage, or simplify model
GPU Not Detected Driver issues or unsupported hardware Update drivers, check hardware compatibility
Poor Performance Suboptimal configuration Adjust batch sizes, check for other GPU-intensive applications
System Crashes GPU overheating or unstable drivers Improve cooling, update/rollback drivers
Training Fails to Start Missing dependencies Check system logs, verify all dependencies are installed
Important: If you experience issues with GPU acceleration, you can always disable it in Settings and fall back to CPU processing, which will work reliably on all systems.
API Reference
Introduction to the API

AITradeStrategist provides a comprehensive REST API that allows you to programmatically interact with the platform. This API enables integration with external systems, automation of tasks, and development of custom extensions.

Authentication

All API requests require authentication using API keys.

Generating API Keys
  1. Go to Settings > API Access
  2. Click "Generate New API Key"
  3. Give your key a descriptive name
  4. Set appropriate permissions
  5. Save and securely store the generated key
Using API Keys

Include your API key in the request header:

Authorization: Bearer YOUR_API_KEY

Rate Limits

To ensure system stability, API requests are subject to rate limiting:

  • Standard Endpoints: 60 requests per minute
  • Resource-Intensive Endpoints: 10 requests per minute
  • Training/Simulation Endpoints: 2 requests per minute

Rate limit information is provided in response headers:

  • X-RateLimit-Limit: Total allowed requests per time window
  • X-RateLimit-Remaining: Remaining requests in current window
  • X-RateLimit-Reset: Time (UTC timestamp) when limit resets
Note: This documentation provides a high-level overview of the API. For detailed parameter information and response formats, refer to the complete API specification available at /api/docs.
Frequently Asked Questions

What is AITradeStrategist?

AITradeStrategist is an advanced cryptocurrency trading platform that leverages machine learning algorithms to optimize trading strategies. It integrates with Freqtrade for execution and provides tools for backtesting, model training, and performance monitoring.

Do I need coding experience to use AITradeStrategist?

No, the platform is designed with a user-friendly interface that doesn't require coding knowledge. However, having some understanding of trading concepts and basic machine learning principles will be helpful.

Is AITradeStrategist suitable for beginners?

While beginners can use the platform, it's recommended to have some experience with cryptocurrency trading. The platform includes educational resources to help new users get started.

How secure is my data?

AITradeStrategist employs industry-standard security practices to protect your data. API keys are encrypted, and the platform doesn't store your exchange credentials unless explicitly configured to do so.

Which AI algorithms does the platform use?

AITradeStrategist primarily uses gradient boosting algorithms like LightGBM and XGBoost, as well as Random Forest. These algorithms are well-suited for time series prediction and pattern recognition in financial data.

How often should I retrain my models?

It depends on market conditions, but generally, retraining every 2-4 weeks is recommended. The platform can also be configured to automatically suggest retraining when model performance degrades.

What's the difference between LightGBM and XGBoost?

Both are gradient boosting frameworks, but LightGBM typically trains faster and handles large datasets more efficiently. XGBoost may provide better accuracy in some cases. The platform allows you to compare both algorithms for your specific trading data.

Can I import my own models?

Yes, AITradeStrategist supports importing externally trained models that are compatible with its framework. This is an advanced feature that requires some technical knowledge.

Do I need to install Freqtrade separately?

Yes, Freqtrade needs to be installed separately. AITradeStrategist connects to your existing Freqtrade installation via its API.

Which version of Freqtrade is supported?

AITradeStrategist supports Freqtrade version 2022.4 and newer. It's recommended to use the latest stable version of Freqtrade for optimal compatibility.

Can I control multiple Freqtrade instances?

Yes, you can configure connections to multiple Freqtrade instances, allowing you to manage different strategies across various accounts or exchanges.

Does AITradeStrategist modify my Freqtrade configuration?

Only if you explicitly allow it to. By default, it only reads data from Freqtrade. If you choose to deploy strategies, it will create strategy files in your Freqtrade directory.

Is a GPU required to use AITradeStrategist?

No, a GPU is optional. All features will work with CPU-only processing, but certain operations like model training will be significantly faster with GPU acceleration.

Why does AITradeStrategist focus on AMD GPUs?

The platform is optimized for AMD GPUs through DirectML integration, providing better performance for these cards. NVIDIA GPUs may work with reduced performance through DirectML compatibility.

How much VRAM do I need?

For basic models, 4GB VRAM is sufficient. For more complex models or larger datasets, 8GB or more is recommended. The platform includes memory management features to work within your GPU's limitations.

I'm getting GPU out-of-memory errors. What should I do?

Try reducing batch sizes, limiting VRAM usage in settings, or simplifying your model. If problems persist, you can disable GPU acceleration and use CPU processing instead.

The application won't start. What should I check?

Verify that all dependencies are installed, check system logs for errors, ensure the database is properly configured, and confirm your hardware meets the minimum requirements.

Model training fails with errors. How can I fix this?

Common causes include insufficient memory, missing dependencies, or data format issues. Check the system logs for specific error messages. Try reducing model complexity or disabling GPU acceleration to troubleshoot.

Can't connect to Freqtrade. What might be wrong?

Verify that your Freqtrade instance is running with API enabled, check network connectivity, ensure API credentials are correct, and confirm that there are no firewall restrictions blocking the connection.

Where can I find detailed error logs?

System logs are available in the Settings > Logs section of the application. For more detailed debugging, check the application log files in the logs directory of your installation.
